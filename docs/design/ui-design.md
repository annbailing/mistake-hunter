# UI/UX 设计文档

> 错题猎人 (Mistake Hunter) | 版本：v1.0 | 日期：2026-05-30

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| 简洁优先 | 学生用户群体，界面力求清爽、无干扰 |
| 信息层次 | 关键信息突出，次要信息弱化 |
| 操作直觉 | 常用操作不超过 2 步完成 |
| 一致性 | 统一的间距、圆角、阴影、交互模式 |
| 深色模式 | 全局支持 Light / Dark 模式切换 |

---

## 六、响应式策略

| 断点 | 宽度 | 布局 |
|------|------|------|
| 手机 | < 768px | 侧边栏隐藏为汉堡菜单；统计卡片单列；表格转为卡片列表 |
| 平板 | 768px - 1024px | 侧边栏收缩为图标模式；统计卡片双列 |
| PC | > 1024px | 完整布局；侧边栏展开；统计卡片四列 |

## 七、交互规范

| 交互 | 规范 |
|------|------|
| 加载 | 骨架屏（Skeleton）代替 Spinner |
| 空状态 | 图标 + 文字提示 + 行动按钮（如"录入第一道错题"） |
| 错误提示 | Toast 通知，3秒自动消失 |
| 表单校验 | 实时校验，错误信息在输入框下方红色显示 |
| 确认弹窗 | 危险操作（删除）使用 Modal 二次确认 |
| 动画 | 过渡时长 200ms，使用 ease-in-out |
