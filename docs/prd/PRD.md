# 错题猎人 (Mistake Hunter) - 产品需求文档

> 版本：v1.0 | 日期：2026-05-30 | 作者：产品经理智能体

---

## 一、产品概述

### 1.1 产品定位

AI 驱动的智能错题管理系统，帮助学生从"记录错题"升级为"消灭弱点"。

### 1.2 目标用户

| 用户角色 | 描述 |
|----------|------|
| 学生（主要用户） | 初高中及大学生，有系统性复习和错题管理需求 |
| 家长（次要用户） | 查看孩子学习薄弱点报告，辅助监督学习 |
| 教师（次要用户） | 查看班级整体错题分布，调整教学重点 |

### 1.3 核心价值主张

- 不只是记录错题，而是**分析错因 + 生成变体题 + 智能复习**，形成完整学习闭环
- AI 驱动的知识薄弱点可视化，让学生知道"该补什么"
- 遗忘曲线提醒，科学安排复习节奏

### 1.4 竞品分析

| 维度 | 传统错题本App | 错题猎人 |
|------|---------------|----------|
| 错题录入 | 手动拍照保存 | 拍照 OCR 自动识别 + 手动输入 |
| 错因分析 | 无 | AI 自动分析错因分类 |
| 变体题 | 无 | AI 自动生成同类变体练习题 |
| 知识地图 | 无 | 可视化知识薄弱点地图 |
| 复习计划 | 固定间隔提醒 | 艾宾浩斯遗忘曲线智能提醒 |
| 进步追踪 | 无 | 多维度进步趋势图 |

## 三、用户故事 (User Stories)

### M1 用户系统

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-1.1 | 作为学生，我想通过手机号注册账号，以便使用错题猎人 | 1. 输入手机号+验证码完成注册 2. 设置昵称和学段（小学/初中/高中/大学） 3. 注册成功自动登录 |
| US-1.2 | 作为学生，我想通过手机号+密码登录 | 1. 输入手机号+密码 2. 支持"记住我" 3. 密码错误3次后显示验证码 |
| US-1.3 | 作为学生，我想编辑个人资料 | 1. 修改昵称、头像、学段、年级 2. 修改密码 |

### M2 错题录入

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-2.1 | 作为学生，我想拍照录入错题 | 1. 支持拍照或从相册选取图片 2. OCR 自动识别图片中的文字内容 3. 识别结果可手动修正 4. 最多支持5张图片 |
| US-2.2 | 作为学生，我想手动输入错题 | 1. 输入题目标题 2. 输入题目内容（支持富文本） 3. 输入我的错误答案 4. 输入正确答案 5. 选择所属科目和章节 |
| US-2.3 | 作为学生，我想给错题添加标签 | 1. 可选已有标签或新建标签 2. 支持多个标签 3. 标签支持自定义颜色 |
| US-2.4 | 作为学生，我想标记错题的来源 | 1. 输入来源（如"月考"、"练习册P23"） 2. 输入来源日期 |

### M3 错题管理

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-3.1 | 作为学生，我想查看错题列表 | 1. 按时间倒序显示 2. 显示题目缩略图、标题、科目、标签 3. 支持分页加载 |
| US-3.2 | 作为学生，我想搜索错题 | 1. 按关键词搜索题目内容 2. 按科目筛选 3. 按标签筛选 4. 按时间范围筛选 5. 按错误类型筛选 |
| US-3.3 | 作为学生，我想编辑已录入的错题 | 1. 修改题目内容、答案、标签等所有字段 2. 保存后更新 |
| US-3.4 | 作为学生，我想删除错题 | 1. 单条删除（二次确认） 2. 批量删除 |
| US-3.5 | 作为学生，我想查看错题详情 | 1. 显示完整题目内容 2. 显示我的错误答案和正确答案 3. 显示 AI 错因分析结果 4. 显示关联的变体题 |

### M4 AI 错因分析

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-4.1 | 作为学生，我想让 AI 分析我为什么做错了 | 1. 自动分析错误原因 2. 分类为：概念混淆、计算失误、审题不清、知识点遗忘、解题方法错误 3. 给出分析说明 4. 分析过程不超过10秒 |
| US-4.2 | 作为学生，我想查看错因分布统计 | 1. 饼图展示各类错因占比 2. 按科目/时间范围筛选 3. 可查看每类错因的具体错题 |

### M5 变体题生成

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-5.1 | 作为学生，我想针对错题生成同类练习题 | 1. 每道错题可生成3道变体题 2. 变体题难度与原题相近 3. 变体题考察同一知识点 4. 提供答案和解析 |
| US-5.2 | 作为学生，我想在线作答变体题 | 1. 显示题目，隐藏答案 2. 输入我的答案 3. 提交后显示正确答案和解析 4. 记录作答结果（对/错） |

### M6 知识薄弱点地图

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-6.1 | 作为学生，我想看到我的知识薄弱点 | 1. 按知识点展示错题数量 2. 颜色深浅表示薄弱程度 3. 点击知识点查看对应错题列表 |
| US-6.2 | 作为学生，我想按科目查看薄弱点 | 1. 按科目维度展示知识点树形结构 2. 每个知识点标注错题数和掌握度 |

### M7 智能复习计划

| ID | 用户故事 | 验收标准 |

| US-7.2 | 作为学生，我想标记错题已掌握 | 1. 复习后标记"已掌握" 2. 已掌握的错题降低复习频率 3. 标记后再次出错则重新进入复习周期 |

| US-8.2 | 作为学生，我想看到今日学习摘要 | 1. 今日新增错题数 2. 今日复习错题数 3. 今日完成变体题数 4. 连续学习天数 |

### M9 科目管理

| ID | 用户故事 | 验收标准 |
|----|----------|----------|

## 四、非功能性需求

| 维度 | 要求 |
|------|------|
| 性能 | 页面加载 < 2秒；OCR 识别 < 5秒；AI 分析 < 10秒 |
| 并发 | 支持 100 并发用户 |
| 安全 | 密码 bcrypt 加密；JWT Token 鉴权；API 输入校验；SQL 注入防护 |
| 兼容性 | 响应式设计，支持 PC + 平板 + 手机浏览器 |
| 可用性 | 界面简洁，符合学生使用习惯，支持深色模式 |

## 五、技术栈建议

| 层 | 技术 |
|----|------|
| 前端 | React 18 + TypeScript + TailwindCSS + Recharts |
| 后端 | Node.js + Express + TypeScript |
| 数据库 | PostgreSQL + Redis（缓存+会话） |
| AI 集成 | 大语言模型 API（错因分析 + 变体题生成） |
| OCR | 百度 OCR API / Tesseract.js |
| 认证 | JWT + bcrypt |

## 六、待澄清问题 (Open Questions)

> 以下问题需要与用户确认后才能推进到下一阶段。

| # | 问题 | 影响范围 | 建议默认值 |
|---|------|----------|------------|
| Q1 | OCR 服务选择：使用免费的 Tesseract.js（本地识别，精度一般）还是付费的百度 OCR API（精度高，需要 API Key）？ | M2 错题录入、M10 题目识别 | 先用 Tesseract.js，后续可切换 |
| Q2 | AI 大模型选择：使用 OpenAI GPT、Claude 还是国产模型（如通义千问、文心一言）？ | M4 错因分析、M5 变体题生成 | 采用可配置的抽象层，支持多模型切换 |
| Q3 | 是否需要多端适配（移动端 App）？ | 整体架构 | V1 先做 Web 版（响应式），后续迭代 App |
| Q4 | 是否需要后端管理后台？ | 整体架构 | V1 暂不做，后续迭代 |
| Q5 | 题目是否需要支持公式/数学符号渲染？ | M2、M5 | 支持 MathJax/KaTeX 渲染 |
